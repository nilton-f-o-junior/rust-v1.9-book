Introduction to Rust

Rust was created by Graydon Hoare at Mozilla Research in 2010. The first stable version, 1.0, was released in 2015, marking the beginning of its popularity and broader adoption in the industry.

Rust is a systems programming language that is compiled and open-source, meaning it is directly translated into machine code, which makes it extremely fast, without the need for an interpreter like Python or JavaScript. It positions itself as an alternative to languages like C and C++, which are known for their performance but are also more susceptible to memory errors, such as null pointers or data races.

Rust's major innovation is the Ownership Model, a system that guarantees memory safety at compile time. This means that the Rust compiler checks if your code is free of memory errors before it's even executed. The compiler becomes a memory "guardian," and if it doesn't approve your code, it won't compile. This approach eliminates an entire class of bugs that are common in other languages.

Comments

Comments are used to document your code, making it more understandable for you and other developers. They are ignored by the compiler, meaning they don't affect how your program runs, only its readability.

  • Clarity: use them to explain something complex or important.
  • Maintenance: make it easier to work with your code in the future.
  • Collaboration: create comments that are understandable to anyone.
  • Debugging: temporarily disable a line or block of code for testing.

// This is a single-line comment.
}

Format

The format! macro in Rust is used to improve data output formatting by using curly braces "{}" as placeholders that will be filled with the information.

For more information, see the Rust std::fmt documentation.

Shadowing

You can declare a new variable with the same name as an existing one, but the previous one will have its value replaced.

#![allow(unused)]
fn main() {
//
let food = "bread";
println!("{}", food);

let food = "milk";
println!("{}", food);

let food = "pizza";
println!("{}", food);
}
#![allow(unused)]
fn main() {
//
let animal = "Cat";
println!("{}", animal);

let animal = "Dog";
println!("{}", animal);

let animal = "Bird";
println!("{}", animal);
}

Scope

Enclosing a block of code within curly braces {} creates a new scope that limits the visibility of variables defined within it. Variables defined in outer sco pes remain accessible, but can be "shadowed" by variables with the same name in the inner scope.

#![allow(unused)]
fn main() {
//
let x = 1;
println!("{}", x);

{
  let x = 2;
  println!("{}", x);
}

println!("{}", x);
}
#![allow(unused)]
fn main() {
//
let name = "Alice";
println!("{}", name);

{
  let name = "Rodolfo";
  println!("{}", name);
}

println!("{}", name);
}

Data Types

This chapter explores the fundamental data types available in Rust, covering the essential building blocks you'll use to store and manipulate information in your programs. Understanding these data types is crucial for writing effective Rust code, as each type has specific characteristics, memory requirements, and use cases.

We'll begin with numeric types, examining both unsigned (u) and signed (i) integers, followed by floating-point numbers for handling decimal values. You'll then learn about Boolean types for logical operations, character types for individual Unicode values, and the important distinction between String and &str for text handling.

The chapter concludes with compound data types—tuples for grouping related values of different types, and arrays for storing multiple elements of the same type. These structures form the foundation for more complex data organization patterns you'll encounter later.

One of Rust's key strengths is its statically typed system, which means the compiler knows the exact type of every variable at compile time. This provides both safety and performance benefits that you'll appreciate as you work through these examples.

By mastering these fundamental data types, you'll gain the knowledge needed to choose the right type for your specific programming needs. As with previous chapters, practice questions are provided at the end to reinforce your understanding and ensure you're ready to move forward confidently.

Operators

This chapter delves into the various operators available in Rust, which are essential tools for performing operations on data and controlling program flow. Operators allow you to manipulate values, make comparisons, and execute logical operations that form the core of any programming logic.

We'll start with arithmetic operators for mathematical calculations, followed by assignment operators for modifying variable values. You'll then explore comparison operators for evaluating relationships between values, and logical operators for combining boolean expressions. The chapter also covers how these operators work specifically with different data types like strings (&str), tuples, and arrays.

Mastering these operators is fundamental to writing expressive and efficient Rust code. They serve as the building blocks for more complex operations and algorithms you'll develop as you progress.

Arithmetic Operators

Used for basic mathematical calculations.

String and &str

This chapter covers Rust's string types – the fundamental tools for handling text data in your programs.

We'll start with string literals: &str for compile-time text, string slices for referencing parts of strings, and their immutable nature. Then we 'll explore owned strings: String for dynamic text creation, push and push_str for building strings, and methods for manipulation and transformation.

We'll also cover conversions – Rust's seamless transitions between &str and String, when to use each type, and how ownership affects your choice.

Each section includes practice questions to reinforce your understanding of these essential text-handling constructs.

Strings

Sequence of UTF-8 characters that can form a word or a phrase, including letters, numbers, symbols, or any other type of special character.

Think of your memory as a cabinet, a variable as a drawer, and a string as a piece of paper.

When you create a "string" variable, you use the entire drawer. But when you create a "&str" variable, you use a piece of paper to write down the information and then put it inside the drawer.

Control Flow

This chapter covers Rust's control flow structures – the fundamental tools for making decisions and repeating actions in your programs.

We'll start with conditional statements: if for basic decisions, else for alternatives, and combined if-else for multiple scenarios. Then we'll explore loops: loop for infinite iterations, while for condition-based repetition, and for for iterating over collections.

We'll also cover match – Rust's powerful pattern matching feature that handles complex conditions with precision and safety.

Each section includes practice questions to reinforce your understanding of these essential programming constructs.

If

The "if" statement works simply: if a condition is met, execute some code.

If 1 + 1 equals 2, print "Hello, world!"

#![allow(unused)]
fn main() {
//
let num_1 = 1;
let num_2 = 1;

if num_1 + num_2 == 2 {
    println!("Hello, world!");
};
}

    println!("Code = {:?} even", code);

}

Match

Match compares a value against different patterns and executes the code corresponding to the first pattern that matches. It's like a more powerful and safer switch.

It ensures that all possible cases are covered, preventing common bugs. Unlike traditional switch, it doesn't need break and always returns a value.

#![allow(unused)]
fn main() {
//
let a = 3;

match a {
  1 => println!("1"),
  2 => println!("2"),
  3 => println!("3"),
  _ => println!("else"),
};
}

range

#![allow(unused)]
fn main() {
//
let age = 63;

let category = match age {
  0..=11 => "child",
  12..=17 => "teenager",
  18..=64 => "adult",
  _ => "elderly",
};

println!("{}", category);
}

println!()

#![allow(unused)]
fn main() {

let alph = 'g';

});
}
